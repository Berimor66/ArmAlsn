===============================================================================
   БИБЛИОТЕКА MICROSOFT FOUNDATION CLASS: обзор проекта ArmAlsn 
===============================================================================

Данное приложение ArmAlsn создано мастером приложений. Это 
приложение показывает основы использования фундаментальных классов Microsoft,
а также является отправной точкой для создания конкретного приложения.

В этом файле содержится краткое описание содержимого файлов, составляющих
конкретное приложение ArmAlsn.

ArmAlsn.vcproj
    Это основной файл проекта для проектов VC++, создаваемых с помощью мастера
    приложений. В нем содержатся сведения о версии Visual C++, создавшей файл,
    а также сведения о платформах, конфигурациях и свойствах проекта, выбранных
    с помощью мастера приложений.

ArmAlsn.h
    Это основной файл заголовка для приложения. В него включены другие
    определенные для проекта заголовки (в том числе Resource.h) и объявлен класс
    приложения CArmAlsnApp.

ArmAlsn.cpp
    Это основной исходный файл приложения, содержащий класс приложения
    CArmAlsnApp.

ArmAlsn.rc
    Это список всех ресурсов Microsoft Windows, используемых программой. В него
    включены значки, рисунки и курсоры, хранящиеся в подкаталоге RES. Этот файл
    можно редактировать непосредственно в Microsoft Visual C++. Ресурсы
    конкретного проекта находятся в 1049.

res\ArmAlsn.ico
    Это файл значка, используемого в качестве значка приложения. Этот значок
    включен посредством основного файла ресурсов ArmAlsn.rc.

res\ArmAlsn.rc2
    Этот файл содержит ресурсы, не редактируемые в Microsoft Visual C++. Все
    ресурсы, не редактируемые редактором ресурсов, следует поместить в этот
    файл.

ArmAlsn.reg
    Это пример REG-файла, показывающий вид настроек регистрации, устанавливаемых
    для вас этой структурой.  Его можно использовать в качестве REG-файла
    для дальнейшей работы с приложением, или просто удалить его и использовать
    регистрацию по умолчанию RegisterShellFileTypes.

ArmAlsn.idl
    Этот файл содержит исходный код языка описания интерфейса для библиотеки
    типов вашего приложения.

/////////////////////////////////////////////////////////////////////////////

Для окна основной рамки:
    В проект включается стандартный интерфейс MFC.

MainFrm.h, MainFrm.cpp
    Эти файлы содержат класс рамок CMainFrame, полученный из
    CFrameWnd и управляющий всеми свойствами рамок SDI.

/////////////////////////////////////////////////////////////////////////////

Мастер приложений создает один тип документов и одно представление:

ArmAlsnDoc.h, ArmAlsnDoc.cpp —  документ
    Эти файлы содержат класс CArmAlsnDoc. Измените эти файлы, чтобы
    добавить особые данные документа и реализовать сохранение и загрузку 
    файлов (с помощью CArmAlsnDoc::Serialize).
    Документ будет содержать следующие строки:
        Расширение файла:          alsn
        Идентификатор типа файла:  ArmAlsn.Document
        Подпись основной рамки:    ArmAlsn
        Имя типа документа:        ArmAlsn
        Имя фильтра:               ArmAlsn Files (*.alsn)
        Новое краткое имя файла:   ArmAlsn
        Длинное имя типа файла:    ArmAlsn.Document

ArmAlsnView.h, ArmAlsnView.cpp — представление документа Эти 
    файлы содержат класс CArmAlsnView.
    Объекты CArmAlsnView используются для представления объектов 
    CArmAlsnDoc.



/////////////////////////////////////////////////////////////////////////////

Мастер приложений создает также классы, специфичные для OLE

CntrItem.h, CntrItem.cpp
    Эти файлы содержат класс CArmAlsnCntrItem. Этот класс 
    используется для управления объектами OLE.  Объекты OLE обычно 
    отображаются классом CArmAlsnView и сериализованы как часть 
    класса CArmAlsnDoc.

SrvrItem.h, SrvrItem.cpp
    Эти файлы содержат CArmAlsnSrvrItem. Этот класс используется 
    для подключения класса CArmAlsnDoc к системе OLE и необязательного
    предоставления ссылок на документ.

IpFrame.h, IpFrame.cpp
    Эти файлы содержат CInPlaceFrame. Этот класс получен из 
    COleIPFrameWnd и управляет всеми свойствами рамки во время активации по 
    месту.

    Проект поддерживает объединенные файлы. В формате объединенного файла 
    документ, содержащий один или более объектов автоматизации, сохраняется в 
    одном файле, но при этом разрешен доступ к его отдельным объектам.

Поддержка справки:

hlp\ArmAlsn.hhp
    Этот файл является файлом проекта справки. Он содержит данные, необходимые 
    для компиляции файлов справки в CHM-файл.

hlp\ArmAlsn.hhc
    Этот файл содержит оглавление проекта справки.

hlp\ArmAlsn.hhk
    Этот файл содержит указатель разделов справки.

hlp\afxcore.htm
    Этот файл содержит стандартные разделы справки для стандартных команд и 
    объектов экрана MFC. Добавьте в этот файл необходимые разделы справки.

hlp\afxprint.htm
    Этот файл содержит разделы справки для команд печати.

makehtmlhelp.bat
    Этот файл используется системой построения для компиляции файлов справки.

hlp\Images\*.gif
    Это файлы точечных рисунков, необходимые для стандартных разделов файла 
    справки для стандартных команд библиотеки Microsoft Foundation Class.

Другие возможности:

Элементы ActiveX
    Приложение включает поддержку использования элементов ActiveX.

Поддержка функций печати и предварительного просмотра

    конкретного приложения.

/////////////////////////////////////////////////////////////////////////////

Прочие стандартные файлы:

StdAfx.h, StdAfx.cpp
    Эти файлы используются для построения файла предкомпилированного 
    заголовка (PCH) с именем ArmAlsn.pch и файла 
    предкомпилированных типов с именем StdAfx.obj.

Resource.h
    Это стандартный файл заголовка, определяющий новые идентификаторы 
    ресурсов. Microsoft Visual C++ прочитывает и обновляет этот файл.

ArmAlsn.manifest
    Файлы манифестов приложений используются Windows XP для описания 
    зависимости приложений от определенных версий параллельных сборок. 
    Загрузчик использует эти сведения для загрузки надлежащей сборки из кэша 
    сборок или закрытого типа сборки из приложения. Манифест приложения может 
    быть включен для распространения в качестве внешнего файла .manifest, 
    устанавливаемого в ту же папку, что и исполняемый файл приложения, или он 
    может быть включен в исполняемый файл в виде ресурса.

Прочие замечания:

Мастер приложений использует "TODO:" для указания частей исходного кода, 
которые следует дополнить или настроить.

Если ваше приложение использует MFC в общей библиотеке DLL, потребуется 
распространить библиотеки DLL для MFC. Если язык вашего приложения отличен от 
языкового стандарта операционной системы, потребуется также распространить 
соответствующие локализованные ресурсы MFC90XXX.DLL. Дополнительные сведения 
по этим двум темам см. в разделе о распространении приложений Visual C++ в 
документации по MSDN.
